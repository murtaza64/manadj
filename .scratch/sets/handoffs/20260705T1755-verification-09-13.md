## What was built

**09 — adjacency click-through.** Rows in the Set pane are clickable
(inner buttons stop propagation). Routing by resolved pin
(`adjacencyView`, so dangling pins go unresolved): Transition pin →
`editor/openPair.ts` request with the uuid; Take pin → existing
`requestTakeReview` flow; unresolved → blank sketch
(`EditorStore.startBlankSketch`, emit-only — opening is not an edit, no
persist). `openPair.ts` mirrors the takeReview one-shot-pending +
CustomEvent pattern; App.tsx listener flips the mode; the mounted editor
consumes, assigns outgoing→A / incoming→B via `assignTrack` (shared-deck
loads, ADR 0022), `loadPair`s, then selects/sketches. `openPair`
re-claims the editor's audible surface so a Conductor started from the
Set pane under a mounted editor stands down on the next adjacency click.

**13 — rehearsal loop.** "⏵ practice" button per adjacency row (works
anywhere the Set pane renders, incl. the Performance view's embedded
library; no mode switch). Stops the Conductor, cues outgoing→A /
incoming→B: positions from `sets/practice.ts` (pure) — planned window →
A at window start − 30s runway (mix→track time via the entry's
native-rate anchor), B at planned entry; unresolved → A at last
(temporally) hot cue else duration − 30s, B at Main cue. Re-press pauses
+ re-cues identically; unloaded tracks cue via a one-shot engine
subscription on decode-ready (re-press mid-load re-parks without
restarting the fetch). Fresh-Take surfacing: `takeSink` invalidates
`['takes']` on persist (new module-scoped `api/queryClient.ts`) and
records latest-per-ordered-pair in `capture/freshTakes.ts`; matching
adjacency shows a "● new take — pin?" chip (latest only, never
auto-pinned); click pins and retires the offer. `TAKE_RECORDED_EVENT` and
TakeHistoryView's listener retired (redundant with sink-side
invalidation).
