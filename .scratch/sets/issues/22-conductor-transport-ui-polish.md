## What to build

The Set toolbar's Conductor transport (▶ Play set / ⏸ / ⏹ / ⤴ Pick up)
grew organically as small inline-styled buttons in the header's right
cluster. Restyle:

1. **Respect the site-wide button style.** Migrate transport, Pick up,
   Auto-fill, and Suggest off hand-rolled inline styles onto the shared

2. **Center the transport.** Play / Pick up / Pause sit centered
   (media-player convention).

## Decisions (triaged 2026-07-05)

- **Centered in the Set pane's header row, not a floating bar** —
  three-zone header: set name + status chips left, transport center,
  Auto-fill/Suggest right. No new floating chrome (it would fight the
  follow-scroll ladder for vertical space).
- **Narrow widths: the transport never shrinks or moves.** It keeps
  center priority; the right-side secondary cluster wraps to a second
  row (or collapses to a ⋯ menu) first.
- **Interaction vocabulary per perf-layout issue 08** (the recently
  hardened app-wide convention): hover = border glow in the control's
  own accent via `currentColor`; press = fill (`--surface1`); on-state =
  color, no glow. No scale effects, no hue-shifts, no bespoke hovers.
  Bright, fully saturated colors per AGENTS.md.
