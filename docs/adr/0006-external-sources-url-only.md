# External Sources are identified by URL only

Audio Provenance must cover audio acquired outside manadj (Beatport purchases, YouTube rips, CD rips). We split Source into Native (integrated: Refresh + download, structured external IDs — SoundCloud) and External (recognized, not integrated), and deliberately rejected per-source ID schemas for External Sources: an unintegrated source's ID has no consumer, while its URL already carries the ID (YouTube video ID, Beatport track ID) in self-describing, clickable, retroactively-parseable form. Provenance for External Sources is therefore an origin label (derived from the URL host, overridable) plus an optional URL; label-only rows cover URL-less origins (cd-rip, unknown). If an External Source is later promoted to Native, existing provenance URLs are parsed into structured IDs then — nothing is lost by having stored only URLs.
