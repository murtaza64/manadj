# Parallel process v2: human review gate, read-only default workspace, merge-based landing, umbrella layout

Amends ADR 0012. A day of multi-agent operation surfaced four failure modes: features landing on `main` un-reviewed, agents clobbering the default workspace's working copy (two same-day strandings via docs edits + post-landing `jj new main`), rebase-landing retry storms when several lanes land at once, and permission prompts from lane workspaces living outside the project root. Decided (grilled 2026-07-05):

1. **Human review gates features.** Bugfixes and incidental maintenance auto-land after agent-owned verification (the standard test suite is a suggestion — the issue's Testing Decisions are the spec; single alembic head is the one hard invariant). Feature work and tracked refactors park at `Status: ready-for-human` with a running lane app and a verification walkthrough; only human approval moves `main`.
2. **The default workspace is read-only for agents.** All agent work — docs included — happens in lanes; the untracked `.lanes/` registry is the only exception. Workspace ownership is explicit (`owner:` session ID in the registry) and transfers only by handover, closing the forked/resumed-session ambiguity.
3. **Landing is merge-based.** A landing merge pins the exact `main` commit verified against; the verified tree lands byte-identical as an ancestor. Retries when `main` moves are new merges — never stack rewrites. Retry invariant: `main` only moves to a descendant of its current position. Rebase remains an intra-lane cleanup tool. Rejected: rebase-landing (verification/review fidelity breaks every time trunk moves mid-land; head-to-head in the process doc's change history), and a landing mutex (stale-lock failure mode; merge retries are cheap enough not to need one).
4. **Umbrella layout** (migration deferred until all lanes idle): `~/manadj/` becomes a plain root holding `default/` (the repo) and one directory per lane, with `.lanes/` at the umbrella root. The umbrella keeps the old repo path, so the opencode project and its session history survive. Rejected: permission-config allowlist alone (fixes prompts, not the tree) and lanes nested gitignored inside the repo (no migration, but leaves workspaces inside a working copy — surprising forever).

Consequences: trunk history is merge-shaped (read linearly with `~merges()` revsets; bisect is noisier — accepted); landing throughput now depends on human review latency for features (deliberate); the process doc `docs/agents/parallel-work.md` is the operational companion to this ADR.
